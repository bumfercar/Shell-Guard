# 쉘 스크립트 사용법

## 📋 프로젝트 설명

**시스템 분석 리포트 생성기** - 리눅스 시스템 정보를 자동으로 수집하고 분석 리포트를 생성하는 쉘 스크립트

### ✨ 쉘 스크립트의 장점 활용:
- **파이프(`|`)**: 여러 명령어를 연결하여 데이터 처리
- **텍스트 처리**: `grep`, `awk`, `sed`로 시스템 정보 추출
- **함수**: 코드 재사용 및 모듈화
- **조건문/반복문**: 디스크 사용량 경고 등
- **자동화**: 한 번의 명령으로 전체 시스템 분석

---

## 🚀 실행 방법

### 1. 스크립트 실행 권한 부여
```bash
chmod +x system_report.sh
```

### 2. 스크립트 실행
```bash
./system_report.sh
```

### 3. 생성된 리포트 확인
```bash
# 리포트 파일이 자동 생성됨 (예: system_report_20251103_082704.txt)
cat system_report_*.txt
```

---

## 📊 수집하는 정보

1. **시스템 기본 정보**
   - OS 종류, 커널 버전, 아키텍처
   - 시스템 가동 시간

2. **하드웨어 정보**
   - CPU 모델 및 코어 수
   - 메모리 사용량 (전체/사용/가용)

3. **디스크 분석**
   - 디스크 사용량
   - 80% 이상 사용 시 자동 경고

4. **프로세스 모니터링**
   - CPU 사용률 높은 프로세스 TOP 10
   - 메모리 사용률 높은 프로세스 TOP 10

5. **네트워크 정보**
   - 네트워크 인터페이스 정보

6. **파일 시스템**
   - 큰 파일 TOP 10

7. **사용자 정보**
   - 현재 로그인된 사용자

## 🎯 실행 결과

### 터미널 출력
- 컬러로 구분된 정보 (정상/경고/에러)
- 리포트 파일명 및 크기
- 시스템 상태 요약

### 생성된 파일
- `system_report_[날짜시간].txt`: 전체 분석 리포트
- 나중에 확인 가능 (보관용)

## 🛠️ 기술적 특징

### 사용된 쉘 기능:
- **Shebang**: `#!/bin/bash` (bash 환경 명시)
- **변수**: 색상, 파일명 등
- **함수**: 코드 모듈화
- **조건문**: `if-then-else`로 경고 판별
- **반복문**: `while read` 디스크 검사
- **파이프**: 명령어 연결 (`ps aux | head`)
- **리다이렉션**: `tee`로 화면+파일 동시 출력
- **명령어 치환**: `$(date)`, `$(hostname)` 등
- **날짜 포맷팅**: `date +%Y%m%d_%H%M%S`

### 사용된 리눅스 명령어:
- `uname`: 시스템 정보
- `uptime`: 가동 시간
- `free`: 메모리 정보
- `df`: 디스크 사용량
- `ps`: 프로세스 정보
- `du`: 파일 크기
- `grep/awk/sed`: 텍스트 처리

이 스크립트를 기반으로 추가 가능한 기능:

## 📞 문제 해결

### "Permission denied" 오류
```bash
chmod +x system_report.sh
```

### 일부 정보가 "가져올 수 없습니다"로 표시
- 정상입니다. Docker 컨테이너 환경에서는 일부 정보에 접근 제한이 있을 수 있습니다.
- 실제 리눅스 서버에서는 모든 정보가 정상적으로 표시됩니다.

### 네트워크 정보가 안 나올 때
- `ip` 또는 `ifconfig` 패키지 설치 필요
- Docker 환경에서는 제한적일 수 있음
